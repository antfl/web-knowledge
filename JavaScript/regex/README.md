# JavaScript 正则表达式

正则表达式（Regular Expression）是用于匹配字符串模式的强大工具。

## 基础语法

### 创建正则表达式

```javascript
// 字面量方式
const regex1 = /pattern/flags;

// 构造函数方式
const regex2 = new RegExp('pattern', 'flags');
```

### 标志（Flags）

- `g` - 全局匹配
- `i` - 忽略大小写
- `m` - 多行模式
- `s` - dotAll 模式（. 匹配换行符）
- `u` - Unicode 模式
- `y` - sticky 模式

## 字符类

### 基本字符类

- `.` - 匹配任意字符（除换行符）
- `\d` - 匹配数字 [0-9]
- `\D` - 匹配非数字 [^0-9]
- `\w` - 匹配字母、数字、下划线 [a-zA-Z0-9_]
- `\W` - 匹配非字母、数字、下划线 [^a-zA-Z0-9_]
- `\s` - 匹配空白字符（空格、制表符、换行符等）
- `\S` - 匹配非空白字符

### 自定义字符类

- `[abc]` - 匹配 a、b 或 c
- `[^abc]` - 匹配除了 a、b、c 的任意字符
- `[a-z]` - 匹配小写字母
- `[A-Z]` - 匹配大写字母
- `[0-9]` - 匹配数字
- `[a-zA-Z0-9]` - 匹配字母和数字

## 量词

- `*` - 匹配 0 次或多次
- `+` - 匹配 1 次或多次
- `?` - 匹配 0 次或 1 次
- `{n}` - 匹配恰好 n 次
- `{n,}` - 匹配至少 n 次
- `{n,m}` - 匹配 n 到 m 次

## 边界

- `^` - 匹配字符串开头
- `$` - 匹配字符串结尾
- `\b` - 匹配单词边界
- `\B` - 匹配非单词边界

## 分组

- `()` - 捕获分组
- `(?:)` - 非捕获分组
- `(?<name>)` - 命名捕获分组

## 反向引用

- `\1`、`\2` - 引用第 1、2 个捕获分组
- `\k<name>` - 引用命名分组

## 前瞻断言

- `(?=...)` - 正向先行断言
- `(?!...)` - 负向先行断言
- `(?<=...)` - 正向后行断言
- `(?<!...)` - 负向后行断言

## 常用方法

### String 方法

- `match()` - 返回匹配结果
- `matchAll()` - 返回所有匹配结果
- `replace()` - 替换匹配内容
- `replaceAll()` - 替换所有匹配内容
- `search()` - 搜索匹配位置
- `split()` - 按匹配分割字符串

### RegExp 方法

- `test()` - 测试是否匹配
- `exec()` - 执行匹配并返回详细信息

## 实际应用

### 验证

- 邮箱验证
- 手机号验证
- 身份证验证
- URL 验证
- 密码强度验证

### 提取

- 提取数字
- 提取邮箱
- 提取 URL
- 提取标签内容

### 替换

- 格式化电话号码
- 隐藏手机号中间四位
- 替换敏感词
- HTML 转义

## 性能优化

- 避免回溯
- 使用非捕获分组
- 避免贪婪匹配
- 使用原子分组
- 预编译正则表达式

## 学习建议

1. 从简单模式开始，逐步学习复杂模式
2. 多做练习，积累常用正则表达式
3. 使用在线工具测试和调试正则表达式
4. 注意性能，避免过度使用复杂正则
5. 结合实际场景应用，加深理解
